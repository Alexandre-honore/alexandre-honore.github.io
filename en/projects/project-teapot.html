Project Teapot

An asymmetrical co-op puzzle game that relies on constant communication. Solve puzzles by making 3D actions affect a shared 2D plane.

Main technical challenge: 2D/3D projection

The core mechanic of Project Teapot, called The Crayon, was the biggest technical challenge of the game. It projects the 3D environment in real time onto the 2D player plane.

In practice, the 3D player manipulates the level’s geometry through translation and rotation, and each move instantly changes the topology seen by the 2D player. This required a robust system that could:

  • Compute collisions between complex 3D objects and a 2D gameplay axis.
  • Handle different material states: solid boxes create projected platforms, while transparent boxes remove obstacles from the 2D plane.
  • Keep physics and visuals perfectly synchronized over the network between both clients.

Concept and art direction

Set in a futuristic scientific complex and guided by a robot, two players test their adaptability. The visual direction emphasizes the duality of the gameplay:

  • 3D world: Low-poly assets with flat shading for strong spatial readability.
  • 2D world: A hand-drawn Paint-like style that contrasts with the clean scientific facility.

A game inspired by Portal, It Takes Two and Split Fiction.

Multiplayer and architecture

  • Network: Client-server architecture with Steam API integration. The room host runs the server.
  • Matchmaking: Direct invitations through the in-game Steam friends list.
  • Communication: Built-in push-to-talk system.
  • Controls: Full keyboard/mouse and controller support, with Steam Input disabled when needed.
