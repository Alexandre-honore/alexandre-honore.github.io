Project

sokIOban

A Sokoban clone in Java Swing where each level demands strategy and reflection. Modular architecture showcasing Model/View separation and asset management.

Overview

sokIOban is a puzzle game inspired by the classic Sokoban, implemented in Java with the Swing interface. The name and visual design are a reference to my It Takes IO project, merging binary signal concepts with box-pushing puzzle mechanics. The goal: push all crates to their designated landing zones while navigating through progressively challenging levels.

Game Mechanics

  • Movement: Arrow keys.
  • Objective: Push crates onto drop zones.
  • Reload: R key to restart the level.
  • Menu: Esc key to return to main menu.
  • Character Selection: Selection menu at startup.

Technical Architecture

Languages and Tools

  • Java: Main language with Swing for UI.
  • Swing: Reactive UI with Observer pattern for refreshing.
  • ImageIO: Loading and managing visual assets.

Modular Structure (by packages)

  • Cell: Cell types (walls, floor, drop zones).
  • Entity: Player, crates, and mobile entities.
  • Vue: Visual components and rendering.
  • Enum: Enumerations (directions, game states).

Key Files

  • MF.java: Main window and orchestration.
  • Menu.java: Menu system (designed via Menu.form).
  • Grid.java: Centralized game logic (grid, levels, cells, entities).
  • Player.java: Player entity and movements.
  • Sound.java: Audio integration management.

Resource Management

  • Levels: Text format stored in levels/ folder.
  • Visuals: PNG images from assets/ folder.
  • Audio: Music files from musics/ folder.
  • Relative paths: Robust resource handling independent of working directory.

Key Learning Points

  • Model/View Architecture: Clear separation of concerns.
  • Asset Management: Dynamic loading of images and resources.
  • Observer Pattern: Reactive synchronization between model and view.
  • Relative Path Resolution: Resource portability.
  • Dynamic Layout: Automatic cell adjustment to window.
  • Form-based UI: Swing Forms for development acceleration.
  • Audio Integration: Basic sound and music management.
  • Text Level Design: Simple and extensible systems.

Specifications

Type: puzzle Language: Java UI: Swing Platform: Desktop Audio: basic

What This Project Demonstrates

  • Modular and scalable Java architecture.
  • Smart asset management and relative path handling.
  • Reactive Swing UI with Observer.
  • Level design and iteration.
  • I/O problem solving and file I/O.
  • Basic audio integration in desktop applications.
