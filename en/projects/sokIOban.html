Project

sokIOban

A Sokoban clone in Java Swing where each level demands strategy and reflection. Modular architecture showcasing Model/View separation.

Overview

sokIOban is a puzzle game inspired by the classic Sokoban, implemented in Java with the Swing interface. The name and visual design are a reference to my It Takes IO. The goal: push all crates to their designated landing zones while navigating through progressively challenging levels.

Game Mechanics

  • Movement: Arrow keys.
  • Objective: Push crates onto drop zones.
  • Reload: R key to restart the level.
  • Menu: Esc key to return to main menu.
  • Character Selection: Selection menu at startup.

Technical Architecture

Languages and Tools

  • Java: Main language with Swing for UI.
  • Swing: Reactive UI with Observer pattern for refreshing.

Modular Structure (by packages)

  • Cell: Cell types (walls, floor, drop zones).
  • Entity: Player, crates, and mobile entities.
  • Vue: Visual components and rendering.
  • Enum: Enumerations (directions, game states).

Key Learning Points

  • Model/View Architecture: Clear separation of concerns.
  • Observer Pattern: Reactive synchronization between model and view.
  • Dynamic Layout: Automatic cell adjustment to window.
  • Form-based UI: Swing Forms for development acceleration.
  • Audio Integration: Basic sound and music management.
  • Text Level Design: Simple and extensible systems.
